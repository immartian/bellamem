# Bellamem v0.2 — Language-Independent Specification

**Status**: reference for the v0.3 TypeScript port.
**Source of truth**: Python `bellamem/proto/*` at tag `v0.2.0-ref`.
**Authors**: v0.2 was designed 2026-04-14 and frozen 2026-04-15.

This document describes what v0.2 IS, independent of Python. A reader
who has never seen the Python code should be able to reimplement v0.2
from this spec plus the JSON schema blocks. Any Python-specific detail
(list vs deque, numpy vs inline cosine) is an implementation choice,
not a spec requirement — unless it's called out as part of an
invariant.

The Python reference lives at `bellamem/proto/`:
- `schema.py`   — Source, Concept, Edge, slugify, mass helpers
- `graph.py`    — Graph container, indices, dedup, stale sweep
- `store.py`    — load/save atomic JSON
- `clients.py`  — Embedder, TurnClassifier, prompts, caches
- `ingest.py`   — read_session_turns, assemble_context, apply, loop
- `walker.py`   — ask_text retrieval + 1-hop edge walk
- `audit.py`    — 5 health signals + thresholds
- `resume.py`   — typed structural summary
- `replay.py`   — chronological session view

Everything else under `bellamem/` (`core/`, `adapters/chat.py`,
`adapters/llm_ew.py`, `bench*.py`, `cmd_*` except the seven v0.2
commands) is dead code and must NOT be ported.

---

## 0. Invariants

These do not change during the port. Violating any of them breaks
dogfood continuity or data compatibility.

1. **Byte-compatible `.graph/v02.json`** — same field names, same
   types, same container shapes (concepts as dict-keyed-by-id, edges
   as list, sources as dict-keyed-by-id). A Python-written graph
   must be readable by the TS port, and vice versa.
2. **Deterministic IDs** — `slugify_topic` produces the same concept
   id for the same topic string, across runs, processes, languages.
   `Edge.id` = first 16 hex chars of `sha256("{type}|{source}|{target}")`.
   `Source.id` = `"{session_id}#{turn_idx}"`.
3. **R1 mass formula is frozen** — `_logit` / `_sigmoid` with
   `MASS_DELTA_NEW_VOICE = 0.5`, `MASS_DELTA_REPEAT_VOICE = 0.1`,
   starting mass 0.5, clamping at 1e-6 / 1-1e-6.
4. **Cache keys are frozen** — `PROMPT_VERSION = "v2"`. Embedder
   cache = sha256(text). Classifier cache = sha256(prompt_version ||
   turn_text || sorted(context_ids) || recent_ids), with `\x00`
   separators between fields. Changing prompt wording or delimiters
   invalidates the whole cache.
5. **Seven ops** — `resume`, `save`, `recall`, `why`, `ask`, `audit`,
   `replay`. Nothing else is in scope for v0.2.
6. **Fail loud, don't cache errors** — classifier errors return a
   synthetic `act=none` result with `was_cached=False`; they must
   NEVER be written back to the cache.

## 1. Data model

### 1.1 Source (immutable)

A Source is an append-only pointer to a speaker turn in a session
jsonl file.

```jsonc
{
  "session_id": "853e838e",           // 8-char prefix of jsonl sessionId
  "file_path": "/abs/path/session.jsonl",
  "speaker": "user" | "assistant",
  "turn_idx": 0,                       // 0-based among *kept* turns
  "text_preview": "first 200 chars",
  "timestamp": 1712864613.105          // POSIX seconds, or null
}
```

- **Key**: `{session_id}#{turn_idx}`. This is the concept `source_refs`
  value and the classifier cache's `recent_ids` entry.
- **text_preview** is 200 chars on disk but the in-memory Source
  object carries up to `MAX_TURN_CHARS = 1500` chars (the classifier
  sees the longer form; the on-disk copy is just for resume/replay
  context).
- `turn_idx` increments only over turns that pass the keep-filter
  (see §3.1). Skipped turns do not consume indices.

### 1.2 Concept (mutable, topic-keyed)

A Concept is a topic-keyed project-model node classified on two axes.

```jsonc
{
  "id": "walker-primitive",            // slugify_topic(topic)
  "topic": "walker primitive",         // human phrase, preserved
  "class": "invariant" | "decision" | "observation" | "ephemeral",
  "nature": "factual" | "normative" | "metaphysical",
  "parent": null | "some-concept-id",  // tree spine (optional)
  "state": null |                      // non-ephemeral → null
           "open" | "consumed" | "retracted" | "stale",
  "mass": 0.5,                         // starts 0.5, R1 accumulates
  "mass_floor": 0.0,                   // reserved, always 0.0 in v0.2
  "voices": ["user", "assistant"],     // ordered, deduped
  "source_refs": ["853e838e#0", ...],  // ordered, deduped
  "first_voiced_at": "853e838e#0",     // source_id of first cite
  "last_touched_at": "853e838e#42"     // source_id of most recent cite
}
```

**Class semantics (temporal profile):**
- `invariant`: time-stable principles or structural facts. Never
  expire. "What the system IS" and "what we commit to" live here.
- `decision`: revisable commitments ("we'll ship X before Y"). May
  be superseded but not automatically aged out.
- `observation`: factual claims about current state ("the bench
  scored N"). May be superseded by a newer observation.
- `ephemeral`: time-bound plans with a state machine. These are the
  only concepts that carry `state`.

**Nature semantics (epistemic type):**
- `factual`: measurable or checkable against reality.
- `normative`: commitments about how we should act or build.
- `metaphysical`: claims about what the system or its concepts ARE.

**Ephemeral state machine:**
```
         ┌── consume-success / consume-failure edge ──→ consumed
open ────┼── retract edge (speaker retraction)  ────→ retracted
         └── R5 sweep (stale after 7 days idle)  ────→ stale
```
Transitions are one-way. Consumed/retracted/stale ephemerals stay
in the graph for audit and history.

**Validation** (enforced at construction time):
- `class` must be one of the four listed values.
- `nature` must be one of the three.
- `state` is non-null IF AND ONLY IF `class == "ephemeral"`. Mint an
  ephemeral without state → default it to `"open"`. Mint any other
  class with a state → error.

**Mass (R1 accumulation):**
```
mass_start              = 0.5          // log-odds 0
MASS_DELTA_NEW_VOICE    = 0.5          // new speaker → +0.5 log-odds
MASS_DELTA_REPEAT_VOICE = 0.1          // repeat speaker → +0.1

logit(p)    = ln(p / (1 - p)), clamped to p ∈ [1e-6, 1 - 1e-6]
sigmoid(x)  = 1 / (1 + e^-x),  clamped at |x| > 30
```
On each call to `cite(source_id, speaker)`:
1. If `source_id` is already in `source_refs`, return immediately
   (idempotent — re-ingesting a turn is a no-op).
2. Append `source_id` to `source_refs`.
3. Update `first_voiced_at` (if null) and `last_touched_at`.
4. If `speaker == ""`, return (pre-R1 code paths can still cite
   without moving mass).
5. If `speaker` is NOT in `voices`: append to `voices`, delta =
   `MASS_DELTA_NEW_VOICE`. Else delta = `MASS_DELTA_REPEAT_VOICE`.
6. `mass = sigmoid(logit(mass) + delta)`.

Mass is bounded in (0, 1) and asymptotically approaches 1 as voices
accumulate. A concept voiced by both user and assistant twice each
sits at roughly mass 0.76.

**Embedding field**: The in-memory Concept also carries a float32
topic embedding for cosine dedup and walker scoring. This is NOT
serialized to `v02.json` — the TS port rehydrates on demand through
the Embedder cache (sha256-of-topic keyed). Loading the graph fresh
leaves every concept with `embedding = null` until first use.

### 1.3 Edge (first-class typed relationship)

```jsonc
{
  "id": "a3f1b2c4d5e6f708",            // sha256("type|source|target")[:16]
  "type": "support" | "dispute" | "cause" | "elaborate" |
          "voice-cross" | "retract" |
          "consume-success" | "consume-failure",
  "source": "853e838e#42" | "walker-primitive",
  "target": "walker-primitive",        // always a concept_id
  "established_at": "853e838e#42",     // source_id that first voiced it
  "voices": ["user"],
  "confidence": "low" | "medium" | "high"
}
```

- **Identity**: `(type, source, target)` triple. Same semantic edge
  voiced twice accumulates voices on the existing edge — R1 applied
  at the edge level.
- **Source**: either a source_id (turn-hub edge, e.g. `consume-success`
  voiced by a turn) or a concept_id (structural concept↔concept edge).
- **Target**: always a concept_id. No edge points at a turn.
- **Confidence bump on merge**: when `add_edge` merges voices into an
  existing edge:
    - `len(voices) >= 2 and confidence == "low"` → `"medium"`
    - `len(voices) >= 3 and confidence != "high"` → `"high"`
- **Edge-type taxonomy**:
    - `support` / `dispute` — epistemic agreement / contradiction
      between concepts (or from a turn to a concept).
    - `cause` — "X because Y", "Y led to X".
    - `elaborate` — "X is a case of Y", "X refines Y".
    - `voice-cross` — neutral mention / ratification edge from a turn.
    - `retract` — speaker-level retraction ("scratch that").
    - `consume-success` / `consume-failure` — ephemeral completion
      events that move the target's `state` to `consumed`.

### 1.4 slugify_topic

```
slug = re.sub(/[^a-z0-9]+/, "-", topic.lower()).strip("-")[:60]
return slug or "unnamed"
```

Lowercase, collapse non-alphanumerics to single dashes, trim edge
dashes, truncate to 60 chars. Empty slugs map to `"unnamed"`. The
TS port's `slugifyTopic` must produce byte-identical output for every
input — this is how cross-session concept identity stays stable.

Collision handling: if `slugify_topic(newTopic)` already exists AND
it's not a dedup hit, the ingest path suffixes with the current
concept count: `slug-{n}` where n = `len(graph.concepts)`. This is
not idempotent across runs and should only fire on true collisions
(different topics producing the same slug). The embedding-based
dedup in §3.3 prevents this in normal operation.

## 2. Graph container

The Graph holds the three primary stores plus derivable indices.

**Primary (serialized):**
```
sources:  { source_id -> Source }      // append-only
concepts: { concept_id -> Concept }    // mutable
edges:    { edge_id -> Edge }          // insert or merge by id
```

**Derived (rebuilt on demand; not serialized):**
```
by_class:        { class_name -> set(concept_id) }
by_nature:       { nature_name -> set(concept_id) }
by_session:      { session_id -> set(concept_id) }
children_of:     { parent_id -> set(child_concept_id) }
open_ephemerals: set(concept_id)
```

**Rebuild trigger**: `Graph.from_json` calls `rebuildIndices()` after
loading. Per-turn ingest keeps indices in sync through `_indexConcept`.
Bulk mutations that bypass those paths must call rebuild before the
next query.

### 2.1 Serialization

```jsonc
{
  "sources":  { "sid#0": {...}, ... },
  "concepts": { "cid1":  {...}, ... },
  "edges":    [ {...}, {...}, ... ],    // list, not dict
  "stats": {
    "n_sources": N,
    "n_concepts": N,
    "n_edges": N,
    "by_class":  { "invariant": N, ... },
    "by_nature": { "factual": N, ... },
    "by_session": { "sid": N, ... },
    "open_ephemerals": N
  }
}
```

`stats` is informational (the reader rebuilds its own counts). Both
Python and TS must round-trip this exact shape.

### 2.2 Atomic save/load

```
save_graph(graph, path):
  ensure parent dir exists
  write JSON (indent=2) to sibling tempfile
  os.rename(tempfile, path)   // atomic on POSIX

load_graph(path):
  if not exists: return empty Graph
  parse JSON
  return Graph.from_json(data)  // rebuilds indices
```

Atomic rename is what keeps a crashed save from corrupting the live
graph. The temp file must live in the same directory as the target
(cross-filesystem rename is not atomic).

### 2.3 find_similar_concept (dedup)

Used at ingest time when `apply_classification` considers creating a
new concept.

```
find_similar_concept(topic, embedding) -> Concept | null:
  slug = slugify_topic(topic)
  if slug in concepts: return concepts[slug]        // canonical hit

  DEDUP_COSINE = 0.78                                // FROZEN threshold
  best = null; best_sim = 0.0
  for c in concepts.values():
    if c.embedding is null: continue
    s = cosine(embedding, c.embedding)
    if s > best_sim: best_sim = s; best = c
  if best and best_sim >= DEDUP_COSINE: return best
  return null
```

`DEDUP_COSINE = 0.78` is frozen for the port. It was lowered from
0.85 after v0.2 diagnostics showed the stricter threshold let ~10%
near-duplicates through. The TS port must use 0.78 so dedup behavior
stays identical across implementations.

### 2.4 nearest_concepts (walker & ingest context)

```
nearest_concepts(query_embedding, k=8, min_sim=0.30) -> [Concept]:
  scored = []
  for c in concepts.values():
    if c.embedding is null: continue
    s = cosine(query_embedding, c.embedding)
    if s >= min_sim: scored.push((s, c))
  scored.sort(desc by s)
  return [c for _, c in scored[:k]]
```

### 2.5 R5 stale sweep

Runs at the end of every `ingest_session`.

```
sweep_stale_ephemerals(now_ts=time.now(), max_age_days=7.0) -> int:
  max_age_sec = max_age_days * 86400
  transitioned = 0
  for c in concepts.values():
    if c.class != "ephemeral" or c.state != "open": continue
    src = sources.get(c.last_touched_at)
    if src is null or src.timestamp is null: continue    // protect pre-ts era
    if (now_ts - src.timestamp) > max_age_sec:
      c.state = "stale"
      open_ephemerals.discard(c.id)
      transitioned += 1
  return transitioned
```

Ephemerals without a timestamped last-touch source are left alone.
This protects legacy graphs from being silently aged out.

### 2.6 rebuild_mass_from_source_refs (repair path)

One-shot R1 repair that resets each concept's `voices`/`mass`/
`first_voiced_at`/`last_touched_at`, then replays stored `source_refs`
through `cite()` with the speaker looked up from `sources`. Used when
a graph was built by a pipeline that wrote `source_refs` directly
without going through `cite()` (e.g. the proto_tree experiments that
preceded v0.2). Needed in the TS port primarily for importing legacy
graphs; day-to-day ingest doesn't call it.

## 3. Ingest

The ingest pipeline converts a Claude Code session jsonl into graph
mutations, one speaker turn at a time.

### 3.1 read_session_turns — streaming jsonl reader

**MUST stream** — large sessions exceed 100 MB and slurp-then-parse
OOM-kills the process.

```
read_session_turns(path, tail=null) -> [Source]:
  session_id = scan_session_id(path)  // first 20 lines, find sessionId field
  container = tail ? bounded_deque(tail) : []
  idx = 0
  for each line in stream(path):
    rec = parse_json(line) ; on error continue
    if rec.type not in ("user", "assistant"): continue
    text = extract_turn_text(rec).strip()
    if empty text: continue
    if text startswith "<" and ">" in text[:120]: continue   // tool notifs
    container.push(Source(
      session_id, path, rec.type, idx, text[:MAX_TURN_CHARS],
      parse_timestamp(rec.timestamp)
    ))
    idx += 1
  return list(container)
```

**session_id derivation**: scan the first 20 parseable lines for a
`sessionId` field (Claude Code puts it on every record). Take first
8 chars. Fall back to `path.stem[:8]` if no sessionId appears.

**extract_turn_text**: `rec.message.content` is either a string or a
list of blocks. For list form, concatenate the `"text"` fields of
blocks with `type == "text"`, joined by newlines. Drop non-text
blocks (tool_use, tool_result).

**parse_timestamp**: accept ISO-8601 with optional trailing `Z`.
Normalize `Z` to `+00:00` before parsing. Return POSIX seconds as
float, or null on any failure.

**tail**: when set, use a bounded collection that keeps only the
last N sources. `turn_idx` still reflects the original stream
position (turns 0..k dropped, turns k+1..N kept with their original
indices intact).

**Filters**:
- Skip tool-use notifications and bracketed system messages (`<...>`
  in the first 120 chars).
- Skip empty turns.
- Skip unparseable lines silently.

### 3.2 Context assembly

Before each turn, assemble the classifier's context:

```
assemble_context(graph, turn, processed_turns, embedder):
  if graph.concepts is non-empty:
    turn_emb = embedder.embed(turn.text[:600])
    nearest = graph.nearest_concepts(turn_emb, k=CONTEXT_K)  // K=8
  else:
    nearest = []
  ephemerals = graph.open_ephemerals_in_session(turn.session_id)
  return (nearest, ephemerals, processed_turns[-RECENT_TURN_N:])  // N=3
```

Constants: `CONTEXT_K = 8`, `RECENT_TURN_N = 3`, `MAX_TURN_CHARS = 1500`.

### 3.3 Classifier prompt contract

The classifier is a single OpenAI chat completion call with
`response_format={"type": "json_object"}` and `temperature=0.0`.

**Model default**: `gpt-4o-mini`. TS port should honor an
`BELLAMEM_MODEL` env override for future model swaps.

**System prompt** (verbatim — changing this requires bumping
`PROMPT_VERSION`):

```
You watch a developer/AI conversation and maintain a project concept
graph for the bellamem project.

For each new turn, decide what the turn does to the graph.

A concept is a distinct IDEA identified by a short topic phrase
(3-10 words) and classified on two axes:

class (temporal profile):
  - invariant: time-stable principles or structural facts; never expire
  - decision: revisable commitments ("we'll ship X before Y")
  - observation: factual claims about current state ("the bench scored N")
  - ephemeral: time-bound plans with a state machine
    (open/consumed/retracted/stale)

nature (epistemic type):
  - factual: measurable or checkable against reality
  - normative: commitments about how we SHOULD act or build
  - metaphysical: claims about what the system or its concepts ARE

The context you receive:
  - nearest_concepts: existing concepts relevant to this turn
  - open_ephemerals: ephemeral plans in this session still "open"
  - recent_turns: the last few turns for anaphora
  - current_turn: the turn to classify

Output strict JSON with:
  - act: "walk" | "add" | "none"
    "walk" = turn reacts to existing concepts
      (ratification, dispute, consume, retract)
    "add"  = turn introduces a genuinely new IDEA
    "none" = procedural (question, meta-auth, ack, tool notification)

  - cites: list of {"concept_id": "<id>", "edge": "<type>"}
    TURN → concept edges. Types: voice-cross | support | dispute |
    retract | consume-success | consume-failure

  - creates: list of {"topic": "...", "class": "...", "nature": "...",
    "parent_hint": "<id|null>"}
    Only for genuinely new IDEAS. Prefer citing existing concepts.

  - concept_edges: list of {"source": "<cid>", "target": "<cid>",
    "type": "<type>", "confidence": "low|medium|high"}
    CONCEPT → concept structural edges. First-class citizens.
    Types: cause | elaborate | dispute | support
      cause:     "X because Y" / "Y led to X"
      elaborate: "X is a case of Y" / "X refines Y"
      dispute:   "X contradicts Y"
      support:   "X confirms Y" (use sparingly)

ONE CONCEPT PER IDEA — do NOT split into attributes.

EXTRACTION RULES:
- Questions → act=none
- Meta-authorization → act=none
- Short acks → act=walk voice-cross IF prior turn had proposal; else none
- Retraction markers ("wait — hold on", "scratch that") → act=walk retract
- Tool notifications, shell output → act=none

CONCEPT HYGIENE:
- Topic phrases: noun-phrase form, concise, canonical
- Prefer merging near-variant topics
- Prefer zero creates over one, one over many
- When in doubt walk vs none, prefer none

EDGE HYGIENE:
- Don't default to "support" for cites; use voice-cross for neutral
- Always scan for cause/elaborate/dispute between touched concepts
- concept_edges is first-class — populate aggressively

Return ONLY valid JSON.
```

**User template**:
```
### nearest_concepts
{nearest_fmt}

### open_ephemerals
{ephemerals_fmt}

### recent_turns
{recent_fmt}

### current_turn
speaker: {speaker}
text: """
{text}
"""

Output JSON only.
```

Concept formatting:
```
- id="..." topic="..." class=... nature=... [state=...]
```

Turn formatting:
```
T{turn_idx} [{speaker}]: {text[:300].replace("\n", " ")}[ …]
```

**Full prompt text must be frozen with `PROMPT_VERSION = "v2"`.**
Any wording change requires bumping `PROMPT_VERSION` to invalidate
the cache. The spec freezes the EXACT system-prompt text in this
document; the TS port should treat this block as data, not as prose
to be reformatted.

Known prompt issues deferred to the port (fix them in the TS
version's `v3` prompt if you bump):
- Retract false-positives on conversational cue words ("stop", "wait",
  "now" without accompanying retraction markers).
- Metaphysical class under-population.
- Over-eager concept minting (surface-word → concept without grounding).

### 3.4 ClassifyResult

```jsonc
{
  "act": "walk" | "add" | "none",
  "cites": [ { "concept_id": "...", "edge": "...",
               "confidence"?: "..." }, ... ],
  "creates": [ { "topic": "...", "class": "...", "nature": "...",
                 "parent_hint"?: "..." }, ... ],
  "concept_edges": [ { "source": "...", "target": "...",
                       "type": "...", "confidence"?: "..." }, ... ]
}
```

A result also carries a `was_cached: boolean` flag (not serialized to
the cache file — applied at read time to distinguish fresh calls from
cache hits for stats).

### 3.5 Classifier cache

```
cache_key = sha256(
  PROMPT_VERSION + "\x00" +
  turn_text + "\x00" +
  ",".join(sorted(context_ids)) + "\x00" +
  ",".join(recent_ids)            // preserved order, NOT sorted
)
```

Storage: `{cache_key -> ClassifyResult JSON}` on disk as a single
JSON object. Loaded on construction, written on `save()`. Writes are
dirty-tracked; `save()` is a no-op if nothing changed.

**Fail-closed rule**: if the OpenAI call raises, return a synthetic
`act=none` result with `was_cached=False` and DO NOT cache it. This
was learned the hard way on 2026-04-14 when a missing `OPENAI_API_KEY`
caused 637 turns to cache as `none`, and the next run happily hit
the poisoned cache instead of retrying.

### 3.6 Embedder

**Model default**: `text-embedding-3-small` (1536 dims).
**Cache key**: `sha256(text)`.
**Storage**: JSON dict `{sha -> float[]}` on disk.
**Write semantics**: dirty-tracked; `save()` is a no-op on unchanged.

Vector ops the port needs:
- `cosine(a, b)` — standard dot / (norm × norm), returns 0 if
  either is zero-length. Inline loop is fine; no SIMD required.
- Float32 storage in memory; JSON cache stores as float64 and the
  reader converts.

### 3.7 apply_classification

Pure function over `(Graph, Source, ClassifyResult, Embedder)`. Side
effects only on the Graph.

```
apply_classification(graph, turn, result, embedder):
  graph.add_source(turn)                        // always, unconditional
  if result.act == "none": return

  // 1) Cites: turn → concept edges
  for cite in result.cites:
    cid = cite.concept_id
    if cid not in graph.concepts: continue
    c = graph.concepts[cid]
    c.cite(turn.id, turn.speaker)
    edge_type = cite.edge or "support"
    try edge = Edge(edge_type, turn.id, cid, turn.id,
                    [turn.speaker], cite.confidence or "medium")
    except invalid edge type: continue          // skip, don't crash
    graph.add_edge(edge)
    // state transitions on ephemerals
    if c.class == "ephemeral":
      if edge_type in ("consume-success", "consume-failure"):
        c.state = "consumed"; open_ephemerals.discard(c.id)
      elif edge_type == "retract":
        c.state = "retracted"; open_ephemerals.discard(c.id)

  // 2) Creates: new concepts with cosine dedup
  for create in result.creates:
    topic = (create.topic or "").strip()
    if empty: continue
    class_ = create.class  in valid set else "observation"
    nature = create.nature in valid set else "factual"
    parent = create.parent_hint if present in graph.concepts else null
    topic_emb = embedder.embed(topic)
    existing = graph.find_similar_concept(topic, topic_emb)
    if existing:
      existing.cite(turn.id, turn.speaker)
      continue
    cid = slugify_topic(topic)
    if cid in graph.concepts: cid = f"{cid}-{len(graph.concepts)}"
    new = Concept(cid, topic, class_, nature, parent, emb=topic_emb)
    new.cite(turn.id, turn.speaker)             // R1 for first cite
    graph.add_concept(new)

  // 3) Concept → concept edges
  for edge in result.concept_edges:
    if edge.source not in graph.concepts: continue
    if edge.target not in graph.concepts: continue
    try e = Edge(edge.type, edge.source, edge.target, turn.id,
                 [turn.speaker], edge.confidence or "medium")
    except invalid edge type: continue
    graph.add_edge(e)
```

Notes:
- `apply_classification` must be deterministic: same (graph, turn,
  result) produces the same mutations.
- New concepts are `cite()`'d after construction (not constructed
  with pre-populated source_refs). This ensures the first citation
  runs the R1 mass path for the first speaker too.
- Invalid edge types are skipped with no error. The classifier is an
  LLM and occasionally emits bad types; crashing the ingest on one
  bad turn is worse than dropping the edge.

### 3.8 ingest_session loop

```
ingest_session(graph, jsonl_path, embedder, classifier,
               on_progress, save_every=25, save_to, tail=null):
  turns = read_session_turns(jsonl_path, tail)
  processed = []
  stats = { total=len(turns), llm_calls=0, cache_hits=0,
            skipped_already_ingested=0,
            act_counts={walk:0, add:0, none:0}, started_at=now }

  for i, turn in enumerate(turns):
    if turn.id in graph.sources:
      stats.skipped_already_ingested += 1
      processed.push(turn); continue            // idempotent re-run
    nearest, ephemerals, recent = assemble_context(graph, turn,
                                                   processed, embedder)
    result = classifier.classify(turn.text, turn.speaker,
               format_concepts(nearest), format_concepts(ephemerals),
               format_turns(recent),
               [c.id for c in nearest+ephemerals],
               [s.id for s in recent])
    if result.was_cached: stats.cache_hits += 1
    else: stats.llm_calls += 1
    stats.act_counts[result.act] += 1
    apply_classification(graph, turn, result, embedder)
    processed.push(turn)

    on_progress(i+1, len(graph.concepts), len(graph.edges),
                stats.llm_calls)                // PER TURN, not gated

    if (i+1) % save_every == 0:
      embedder.save(); classifier.save()
      if save_to: save_graph(graph, save_to)

  embedder.save(); classifier.save()
  stats.stale_transitions = graph.sweep_stale_ephemerals()
  stats.finished_at = now
  stats.elapsed_s = finished_at - started_at
  return stats
```

Progress callback fires per turn, not every `save_every` turns. The
earlier gated behavior made cold-cache first-run ingest look like a
hang for the first 25 turns.

Incremental ingest is idempotent: re-running against the same session
skips turns already in `graph.sources`. This makes the dogfood cron
cheap — repeated runs only process newly appended turns.

## 4. Walker (ask/recall/why)

`ask_text(graph, focus)` renders a relevance-first pack: seed concepts
matched on substring-or-cosine, scored by `max(substring, cosine) ×
mass`, followed by a 1-hop edge walk.

### 4.1 Scoring

```
tokenize(q)                   // lowercase, [a-z0-9]+, len >= 2
substring_score(text, tokens) // hits / len(tokens), 0 if no tokens
cosine(a, b)                  // standard, 0 if either is zero-length

score(concept, q_tokens, q_emb):
  sub = substring_score(concept.topic, q_tokens)
  cos = (q_emb && concept.embedding) ? cosine(q_emb, concept.embedding) : 0
  base = max(sub, cos)
  if base <= 0: return 0
  return base * concept.mass
```

**Why max(substring, cosine) not sum**: substring ensures literal
queries work even when the embedder is offline or a concept's
embedding never hydrated; cosine catches semantic matches. Pure
cosine would drop "github pages" off a "docs site publishing"
concept; pure substring would miss it. Max lets either path promote
the concept.

**Why × mass**: a ratified decision at cosine 0.6 should outrank a
one-off mention at cosine 0.7.

### 4.2 Seed + walk

```
ask_text(graph, focus, embedder, seed_k=12,
         class_filter=null, min_score=0.05):
  if empty(graph.concepts): return "empty graph — run bellamem save"

  q_tokens = tokenize(focus)
  q_emb = null
  if embedder and focus.strip():
    try q_emb = embedder.embed(focus)
    // hydrate concept embeddings on demand (cache hits are free)
    if q_emb:
      for c in graph.concepts.values():
        if c.embedding is null:
          try c.embedding = embedder.embed(c.topic) ; except: pass

  // rank
  scored = []
  for c in graph.concepts.values():
    if class_filter and c.class != class_filter: continue
    s = score(c, q_tokens, q_emb)
    if s >= min_score: scored.push((s, c))
  scored.sort desc by s
  seeds = scored[:seed_k]
  seed_ids = set(c.id for _, c in seeds)

  // one-hop edge walk
  neighbors = {}   // { edge_type -> [(src_topic, tgt_topic, dir)] }
  for e in graph.edges.values():
    if e.source in seed_ids:
      ... push (src.topic, tgt.topic or "(turn ...)", "out")
    elif e.target in seed_ids:
      ... push (src.topic or "(turn ...)", tgt.topic, "in")

  render sections in priority order:
    1. dispute    (⊥ disputes)
    2. retract    (retracts)
    3. cause      (causes)
    4. support    (supports)
    5. elaborate  (elaborates)
```

Deduplicate neighbors by `(src_topic, tgt_topic)` within a section.

### 4.3 Output format

```
# v0.2 ask (focus: 'walker primitive')
  558 concepts · 1151 edges · 42 candidates · mode=cosine+mass

## relevant concepts (12)
  s=0.72 m=0.85 [invariant/metaphysical] walker primitive
  ...

## ⊥ disputes (3)
  concept A  ⊥  concept B
  ...

## causes (5)
  concept X  ⇒  concept Y
  ...

— 12 seeds, 20 edge neighbors —
```

Mode is `cosine+mass` when query embedding is available, else
`substring+mass`.

Arrows: `⊥` for dispute, `retract` for retract, `⇒` for cause, `+`
for support, `→` for elaborate.

### 4.4 `recall` vs `ask` vs `why`

In v0.2 these are the same underlying call with different defaults
or framings; the TS CLI should dispatch them to `ask_text` directly.
`why` additionally surfaces the cause-edge layer more prominently in
the render. Exact framing is a CLI-level concern, not a spec concern —
the underlying walker is one function.

## 5. Audit

Five health signals over the current graph state.

### 5.1 Signals

```
concept_density    = n_concepts / n_sources
structural_ratio   = structural_edges / all_edges
                     (structural = cause|elaborate|dispute|support
                      with both endpoints as concepts)
mass_floor_fraction = count(c.mass <= 0.501) / n_concepts
mass_spread        = normalized bucket-entropy (see below)
orphan_refs        = count of source_refs pointing at missing sources
```

**mass_spread**: quantize each concept's mass into 10 buckets over
[0.5, 1.0] (`idx = clamp(floor((mass - 0.5) * 20), 0, 9)`), compute
Shannon entropy of the bucket populations, normalize by `log(10)`.
Result in `[0, 1]`. Higher = better discrimination.

### 5.2 Thresholds (frozen)

```
DENSITY_SOFT     = 0.60    // >0.6 concepts per source = explosion
DENSITY_HARD     = 0.85
STRUCTURAL_SOFT  = 0.10    // <10% cc-edges = thin structural layer
STRUCTURAL_HARD  = 0.04
FLOOR_SOFT       = 0.20    // >20% at floor = R1 starving
FLOOR_HARD       = 0.50
SPREAD_SOFT      = 0.20    // lower = worse here
SPREAD_HARD      = 0.08
orphan_refs      = any > 0 is HARD
```

Each signal maps to `"ok" | "soft" | "hard"` via:
```
verdict_high_is_bad(v):  v >= hard ? "hard" : v >= soft ? "soft" : "ok"
verdict_high_is_good(v): v <= hard ? "hard" : v <= soft ? "soft" : "ok"
```

### 5.3 Ephemeral health

Count of ephemeral concepts by state:
```
{ open: N, consumed: N, retracted: N, stale: N, none: N }
```
`none` catches any ephemeral with a missing/invalid state. The
resume renderer surfaces open/consumed/retracted/stale; growing
`open` with stagnant `consumed` flags missing completion events.

### 5.4 AuditReport output

```
AuditReport {
  n_concepts, n_edges, n_sources,
  signals: [Signal {name, value, verdict, note}, ...],
  ephemeral: { open, consumed, retracted, stale, none }
}
```

`format_audit(report)` renders a human-readable block for inclusion
in the resume output. Only red flags (`soft`/`hard`) are surfaced by
resume; `bellamem audit` shows the full report.

## 6. Resume

`resume_text(graph)` renders the typed, structural summary used by
`/bellamem resume`. This is the context-reconstitution entry point —
it replaces flat memory-file handoff.

### 6.1 Section order

```
1. invariant × metaphysical   — what the system IS
2. invariant × normative      — what we commit to
3. invariant × factual        — structural facts
4. ephemeral × open           — work in progress (last_touched desc)
5. ephemeral × retracted      — rejected approaches (last_touched desc)
6. decision (all natures)     — recent decisions (last_touched desc)
7. dispute edges              — live contradictions
8. retract edges              — speaker retractions
9. stats footer               — by_class, by_nature, ephemeral states,
                                 edge_types
10. audit red flags           — soft/hard only, omit if none
```

### 6.2 Ranking within sections

- **Invariants**: by `mass` desc, tie-break by `len(source_refs)` desc.
- **Ephemerals + decisions**: by `last_touched_at` desc (lexicographic
  string compare works because source_ids are `{session}#{idx}` and
  within a session the `idx` suffix sorts numerically when zero-padded
  — v0.2 does NOT zero-pad so comparisons are best-effort; the TS
  port should preserve the same behavior for diff parity rather than
  "fixing" it).
- **Disputes/retracts**: first N edges in insertion order (no rank).

### 6.3 Counts & defaults

```
top_invariant_meta  = 12
top_invariant_norm  = 10
top_invariant_fact  = 6
top_open_ephemeral  = 15
top_retracted       = 10
top_decision        = 12
top_dispute_edges   = 10
top_retract_edges   = 8
```

### 6.4 Line format

```
  m={mass:.2f} [{source_refs_count:2}r] {topic}
  [open] {topic}
  [retracted] {topic}
  [decision/{nature}] {topic}
  {src_topic}  ⊥  {tgt_topic}
  (turn {source_id})  ⊥  {tgt_topic}       // when src isn't a concept
  (turn {source_id})  retract  {tgt_topic}
```

The leading two-space indent on content lines is load-bearing — the
resume output is read by the slash-command wrapper and section
headers start with `##`, so content lines must start with whitespace
to avoid being parsed as headers by a downstream markdown renderer.

## 7. Replay

`replay_text(graph, session=null, since_turn=0, max_lines=120,
preview_chars=140)` renders a chronological view of a session.

### 7.1 Session picker

If `session` is explicitly set, use it (error if not found).

Otherwise pick "the session I'm in right now":
1. Build `by_session_ts`: max timestamp per session (skipping
   timestamp-less sources).
2. Build `by_session_idx`: max turn_idx per session.
3. If any timestamped sessions exist, return the one with the
   greatest max-timestamp.
4. Else fall back to the session with the greatest max-turn-idx.
5. Else return null (empty graph).

**Why timestamp-over-length**: the earlier "longest session wins"
picker surfaced yesterday's 800-turn session over today's 60-turn
session. Wall-clock is the honest signal.

### 7.2 Render

```
# v0.2 replay (session: {sid})
  {n_turns} turns · {n_concepts} concepts · {n_edges} edges

  (dropped {n} head turns — showing tail {max_lines})    // if tail

#{turn_idx:4d} [{speaker:9}] {text[:preview_chars]}[ …]
          └─ {cls3}/{nat3}  {concept_topic}               // up to 4
          └─ … +{n} more

— {n_shown}/{total} turns shown —
```

Tail-preserve behavior: when `total > max_lines`, drop head turns and
keep the most recent `max_lines`. The replay should always end at
"now".

`{cls3}` and `{nat3}` are the first 3 chars of the class and nature
strings (`inv/met`, `dec/nor`, etc.).

## 8. CLI surface

v0.2 has exactly seven ops. The TS port implements them as Commander
(or equivalent) subcommands on a single `bellamem` binary.

```
bellamem resume [--graph PATH]
bellamem save   [--tail N]
bellamem recall <query> [--class CLASS]
bellamem why    <topic>
bellamem ask    <query>
bellamem audit
bellamem replay [--session SID] [--since N] [--max-lines N]
```

Plus:
```
bellamem-guard   (PreToolUse hook, stdin JSON protocol)
```

**Ops not in v0.2** (do not implement, do not port tests for):
`expand`, `emerge`, `before-edit`, `prune`, `decay`, `scrub`,
`surprises`, `entities`, `render`, `bench`, `ingest-cc`.

### 8.1 Per-project lock

`bellamem save` takes a per-project flock before mutating the graph
file. Lock path is derived from the project root (the directory
containing `.graph/`). The lock PREVENTS the dogfood cron and an
interactive `/bellamem save` from racing on the same project — not
a global `/tmp` lock (that was a v0.2.1 bug that serialized
unrelated projects).

Locking is best-effort advisory (`flock`/equivalent on the lock
file); cross-platform libraries: Node's `proper-lockfile` or the
equivalent.

### 8.2 Env resolution (load order)

```
1. Process environment (shell env wins)
2. $CWD/.env
3. XDG user config .env   (platformdirs / os.homedir-based)
```

`setdefault` semantics — first loader to set a key wins, so shell
env takes precedence, then cwd, then user config.

### 8.3 Claude Code project-dir encoding

The guard hook resolves the path to `.graph/v02.json` for the
*target* project (the one running the tool call), which is encoded
in Claude Code's project-dir convention (hyphenated absolute path
under `~/.claude/projects/`). The TS port must preserve this
encoding so a Node-installed guard can still find the graph for a
Python-era project directory.

## 9. Diff harness (Phase 1 acceptance)

Before declaring the TS port done:

```
scripts/diff-python-vs-ts.sh <command> <args...>
```

Runs the same command against the same `.graph/v02.json` with both
Python (`v0.2.0-ref` tag) and TS, and diffs stdout. Used for every
subcommand in §8. Any drift is a bug in the port.

Specific invariants checked by the harness:
1. `resume` output is byte-identical modulo trailing whitespace.
2. `save --tail 50` on a fresh classifier cache produces the same
   `by_class`/`by_nature` distributions ±1 concept.
3. `recall <query>` returns the same seed concept order for 10
   canonical queries.
4. `audit` returns the same signal values to 3 decimal places.
5. `replay` produces the same turn ordering and concept attachments.

The classifier cache is the source of non-determinism: TS and Python
hit the cache for the same (prompt_version, turn_text, context_ids,
recent_ids) and return byte-identical classifications, so a warmed
cache makes the diff harness reliable.

## 10. What the port must NOT do

- Extend the schema (no new fields, no new classes/natures/states).
- Change `DEDUP_COSINE`, `MASS_DELTA_*`, `PROMPT_VERSION`, or any
  threshold in §5.
- Move to a new on-disk format (e.g. SQLite, message pack) — the port
  reads and writes `.graph/v02.json` and nothing else for v0.3.0.
- Add features from the "Phase 3 — Native Node wins" list
  (localhost web UI, guard daemon, progressive walker). Those land
  AFTER the diff harness passes.
- Fix prompt issues (retract false positives, metaphysical
  under-population). These wait for `PROMPT_VERSION = "v3"` which is
  a separate change post-port.
- Change any visible output string that the diff harness compares.
  If you have a better render, land it in a v0.3.x minor after the
  port's diff harness is retired.

## Appendix B: JSON cache formats

Embedder cache (`proto-embed-cache.json`):
```jsonc

```

Classifier cache (`proto-llm-cache.json`):
